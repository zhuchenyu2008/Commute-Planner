# 行程详情页面规格

## 页面目标

详情页展示单次行程的完整执行状态：锁定路线、最晚出发时间、分段路线、自动复算、提醒计划和用户可执行操作。它是用户确认“什么时候出门、按哪条路线走、提醒是否已设置”的主要页面。

## 用户任务

- 查看目的地、到达时间和准点风险。
- 理解当前锁定路线及总耗时。
- 查看每段行程和缓冲时间。
- 手动复算当前路线。
- 调整缓冲或更改路线。
- 停止监控并取消后续提醒。

## 全局约定

- 未登录访问详情页时跳转登录门禁，登录成功后回到原行程详情。
- 详情页不作为底部主 tab，可从首页当前行程或历史记录进入。
- 页面顶部提供返回按钮；移动端返回到来源页，来源缺失时回首页。
- 手动复算默认不改变锁定路线，除非用户明确选择“更改路线”。
- 视觉沿用现有行程规划详情样板：清晰标题、路线摘要、小地图、纵向分段时间线、底部主操作。

## 布局结构

1. 顶部导航
   - 返回按钮。
   - 目的地名称。
   - 右侧可保留更多操作入口。

2. 行程状态摘要
   - 到达时间。
   - 准点状态：正常、延误风险、需要立刻出发、已取消、已完成。
   - 状态点使用成功、警告、错误三类颜色。

3. 已选方案区
   - 标题：“已选方案”。
   - 路线名称，例如“最快路线 · 地铁 + 共享单车”。
   - 总耗时、最晚出发时间、预计到达时间。
   - 地图预览。
   - “查看实时地图”按钮预留。

4. 行程拆解区
   - 纵向时间线展示每段路线。
   - 分段类型包括：步行、公交、地铁、骑行、驾车、场内步行、等待/换乘缓冲。
   - 每段展示耗时、起终点、线路号/站数/出口等可用信息。

5. 自动复算区
   - 显示是否开启。
   - 显示上次检查时间、下次检查时间、变化通知阈值。
   - 文案示例：“自动复算已开启 · 出发前按 10/5 分钟节奏检查”。

6. 提醒计划区
   - 展示 T-30、T-20、T-15、T-10、T-5、T 的计划提醒。
   - 已触发、待触发、已取消用不同状态区分。
   - 同时标明 Telegram 和邮件是否启用。

7. 操作区
   - 主按钮：出发。
   - 次按钮：手动复算、调整缓冲、更改路线。
   - 危险操作：停止监控。

## 页面状态

- `unauthenticated`：跳转登录页。
- `loading`：加载行程详情。
- `notFound`：行程不存在或已被清理。
- `activeOnTime`：行程正常。
- `activeRisk`：存在延误风险，但尚未到必须出发时间。
- `departNow`：已到或超过最晚出发时间。
- `rechecking`：正在复算锁定路线。
- `recheckFailed`：复算失败，保留旧路线和旧提醒状态。
- `cancelled`：监控已停止，提醒已取消。
- `completed`：行程已结束。
- `authExpired`：认证过期，跳转登录页。

## 核心交互

- 页面加载时调用 `GET /api/trips/{id}`。
- 点击“手动复算”调用 `POST /api/trips/{id}/recheck`，只复算当前锁定路线。
- 复算后如果最晚出发时间变化，页面更新摘要、提醒计划和自动复算状态。
- 点击“调整缓冲”打开底部抽屉，用户修改场内步行、换乘、停车等分钟数后调用 `PATCH /api/trips/{id}/buffer`。
- 点击“更改路线”打开候选路线抽屉，用户确认后调用 `PATCH /api/trips/{id}/route`，并重新生成提醒计划。
- 点击“停止监控”必须二次确认，确认后调用 `POST /api/trips/{id}/cancel`。
- 已取消或已完成行程不再显示“出发”和“手动复算”为主操作。

## 接口

### `GET /api/trips/{id}`

返回行程摘要、锁定路线、分段、提醒计划和通知状态。

### `POST /api/trips/{id}/recheck`

复算当前锁定路线，不自动切换路线。

### `POST /api/trips/{id}/cancel`

取消自动复算和后续提醒。

### `PATCH /api/trips/{id}/buffer`

请求示例：

```json
{
  "insideVenueMinutes": 10,
  "waitAndFrictionMinutes": 8
}
```

### `PATCH /api/trips/{id}/route`

请求示例：

```json
{
  "chosenPlanKey": "habit:柳西"
}
```

## 验收点

- 详情页可以从首页当前行程和历史记录进入。
- 页面明确展示锁定路线，用户不会误以为复算会自动换路线。
- 手动复算失败时旧数据仍然可见，并显示可重试提示。
- 停止监控后提醒计划展示为已取消，不再显示待触发提醒。
- 调整缓冲后最晚出发时间和提醒计划同步更新。
